About this Application File

Wallpaper Slideshow Portal is an Android application source code file. This application lets you to view the images in your device as slideshow wallpaper. It is automatically placed in the Live wallpapers section. The application is packed with a variety of settings such as slide duration, scrollable images, recursive folder and more. This application has very low CPU usage and so the device performance is not affected.

Contents in Source Folder

After purchasing the application source code and extracting, you will find the following folders:

  • Install APK 
  • Project
  • Readme

Install APK: Contains the WallpaperSlideshowPortal.apk application file. You can test run it in your Android device.

Project: Contains the application source code which you have to import into Eclipse and build the application.

Readme: An elaborate readme guide with instruction on how to use this file to successfully build your Android applications.

Getting Started

Before starting to use this source code and developing app you need to first install the following softwares to your computer. If you have already installed these softwares you can skip this step.

1. JDK 5 or above: Install Java Development Kit (JDK) 5 or greater. You can download here. 

2. Eclipse IDE: You can choose Eclipse Indigo or other packages. I personally prefer Indigo. You can download here. Extract the zip file into a folder for example D:\ANDROID\Eclipse after downloading. Run eclipse.exe. On first time you will be asked for workspace directory. It is a directory where you want your android project to be stored. For now just create new folder Workspace in D:\ANDROID or any other directory as you wish.

3. Android SDK: Install android sdk. You can download here. Now you need to connect Eclipse and Android sdk. On Eclipse GUI, select Windows > Preferences menu on top. Select Android and then locate your SDK Location, on my machine it is D:\ANDROID\android-sdk, and then click Apply.

5. Environment Variables:

  • Go to Control Panel > System > Advanced > Environment Variables (or) Right click My Computer > Advanced > Environment Variables.
  • Add the JDK path in PATH environmental variable on your system.
  • Add the SDK tools path (e.g.D:\android-sdk\tools in my case) in the same PATH environmental variable. Click OK to save changes.

6. ADT Plugin for Eclipse: In Eclipse go to Help > Install New Software. Click Add button. On Add Repository window enter some name (e.g.Developer Tools) and enter location as https://dl-ssl.google.com/android/eclipse/ and then click OK. Give check to Developer Tools, then click Next button. Follow the instruction till finish.

Importing Project into Eclipse

1. Extract the downloaded archive that you have purchased in Envato.

2. Open Eclipse > Select File > Import.

3. In the Import window page select Existing Projects into Workspace under General and click Next. In the next window choose Select archive file. Press the Browse button and choose wallpaperportal.zip file present in the Project folder. Click Finish.

4. Your project will be visible on the Package Explorer.

At the left side you can see the project folder. You can edit your files from here.

Changing Application Name

Go to res/values/ directory and edit strings.xml file.

To change App name: Enter the name in the below string.

<string name="app_name">Wallpaper Slideshow Portal</string> <!-- Application Name -->

 

Changing Application Icon

The application icon is present at res/drawable-(dpi)/ic_launcher.png. The icon is placed for ldpi, mdpi, hdpi and xhdpi.

Here the icon sizes:

  • ldpi - 36px x 36px
  • mdpi - 48px x 48px
  • hdpi - 72px x 72px
  • xhdpi - 96px x 96px

Instead you can place only one image in drawable/ folder alone and leave other dpi specific drawable folders. But that's not the best way. I prefer placing icons matching different dpi screens.

Create your icon image with the same file name and replace the existing file. Refresh the project for changes to take effect.

Changing Live Wallpaper Thumbnail

In your Android device you can find live wallpapers though... Menu > Wallpaper > Live Wallpapers.

After installing this application you can see this application thumbnail in Live wallpapers section, as shown below.

The application thumbnail is present at res/drawable/livethumb.jpg. Replace it with any image of your choice. It's better to use image with small file size so that it loads up quickly in live wallpaper section.

 

Compiling the Application

After customizing this app, you need to compile into an .apk file that can be installed in Android devices. Here are the steps of compiling this application.

1. Select the WallpaperPortal project in Package Explorer, Right click and select Export.

2. In Export window select Android > Export Android Application. Then, click Next button.

3. Your project name will appear in Project: box. If not, click Browse and find. Click Next button to proceed.

4. Select your keystore or create a new one. In Location, click Browse button to decide keystore name and the location of it. Give a strong password and proceed to next step.

5. Fill the key creation form and press Next.

6. Finally, click Browse button to specify the location where you want to have to the generated apk, and then click Finish. Now your application is ready. Install it. Have fun.

Troubleshooting

I have checked the application several times and it's absolutely free from errors.

In case you see any errors, do the following:

  • Select the project
  • Click Project > Clean..

Few users may get this error while importing the application:

Android requires compiler compliance level 5.0 or 6.0. Found '1.7' instead.

Android requires compiler compliance level 5.0 or 6.0. Found '1.5' instead.

Or some similar version error.

This is a very common problem with Android development in Eclipse platform.

To fix this error:

  • Clean your project and build it again (or)
  • Right click Project > Select Android Tools > Select Fix Project Properties (or)
  • Right click Project > Select Properties > Select Java Compiler, check Enable project specific settings and select 1.5 or 1.6 from Compiler compliance settings select box. Click Apply and rebuild the project.
